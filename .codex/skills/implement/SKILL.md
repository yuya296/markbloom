# Implement Skill

## 目的

実装開始時の抜け漏れを防ぎ、着手直後に必要な判断を揃える。

## 手順

1. 現在の作業状態を確認する。
2. ブランチが未作成なら、リポジトリ規約に従って `feature/*` / `fix/*` / `docs/*` / `ci/*` を作成する。
3. `AGENTS.md` を読み、Definition of Done と Change Checklist を先に固定する。
4. 変更対象のモジュールを 1-3 個に絞り、変更しない範囲も明記する。
5. 実装前に以下を1メッセージで共有してからコード変更に入る。
   - 目的
   - 変更範囲
   - 検証方針
6. 実装中は小さく区切って進め、各区切りで差分意図を説明する。

## 実装着手テンプレート

```md
実装方針:
- 目的:
- 変更範囲:
- 非対象:
- 主要リスク:
- 検証:
```

## 完了前チェック（このスキルで事前に宣言）

- ローカル検証で実行するコマンドを事前に列挙する。
- 仕様が曖昧な点は推測実装せず、仮定として明示する。
